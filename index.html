Hora do QA - Métodos HTTP

Entenda os principais métodos utilizados em APIs REST

GET
Recupera informações de um recurso sem alterar dados no servidor.
GET /usuarios/123
Consulta Seguro Cacheável
POST
Cria novos recursos ou envia dados para processamento.
POST /usuarios
Criação Envio de Dados
PUT
Atualiza completamente um recurso existente.
PUT /usuarios/123
Atualização Total Idempotente
PATCH
Atualiza apenas os campos informados do recurso.
PATCH /usuarios/123
Atualização Parcial Performance
DELETE
Remove um recurso do sistema.
DELETE /usuarios/123
Exclusão Idempotente
HEAD
Retorna apenas os cabeçalhos da resposta, sem o corpo.
HEAD /arquivo.pdf
Metadados Validação
OPTIONS
Informa quais métodos HTTP são suportados por um recurso.
OPTIONS /usuarios
CORS Descoberta
TRACE
Utilizado para diagnóstico do caminho percorrido pela requisição.
TRACE /api
Debug Diagnóstico
CONNECT
Cria um túnel de comunicação com o servidor, geralmente para HTTPS.
CONNECT api.exemplo.com:443
Proxy HTTPS
